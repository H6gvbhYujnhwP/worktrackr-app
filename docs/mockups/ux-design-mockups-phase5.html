Decisions needed before the build (my recommendation in each)

  1. Recurring paid-gating model. WorkTrackr isn't the accounts package (§9), so there's no per-month "paid" flag for contracts. Recommend: accrue one month of recurring commission per Active contract per commission period; the gate is contract status active + the existing manager per-period approval. This matches §7.2 "while … the customer remains active." Alternative: a manager "collected this period" toggle per contract (more accurate, much more admin).
  2. Annual line normalisation. Recommend: store each line at its native interval (monthly/annual); annual ÷ 12 for the monthly-profit figure and the commission basis. Shown as /yr ÷12.
  3. Mixed quotes (one-off + recurring lines). Recommend: a contract takes only the recurring lines; offer a one-click "create a linked Order for the one-off lines" (or ignore). Confirm that's the right split vs. forcing both onto one record.
  4. Mid-period start / cancel. Recommend v1: count a full month if the contract is Active at the period boundary; no pro-rating (a later enhancement). OK?
  5. Bonus threshold. Should recurring monthly charge count toward the personal performance-bonus turnover threshold? Recommend: yes (it's turnover they generated).
  6. Primary entry point. Recommend: a new Contracts item in Sales (list + form, quote-driven), plus "New contract" on the company profile; with the "act on quote" panel (screen 14) as a secondary entry built in a later batch. OK?
  7. Company-profile "Monthly profit" card source. Recommend: compute it from active contracts (replaces the manual crm.totalProfit field as the displayed figure). Keep the manual field as an optional manual override, or retire it? Your call.
